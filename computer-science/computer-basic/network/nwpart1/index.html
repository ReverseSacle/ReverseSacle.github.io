# 网络、互连网(互联网)和因特网

# ! 网络

  • 由若干结点(Node)和连接这些结点的链路(Link)组成
  • 设备和将设备连接的交换机也是结点
  • 结点之间的互连需要使用链路(有线链路和无线链路)
  • 多个网络通过路由器互连，扩大了网络范围，即互联网(网络中的网络)
  • 最大的互联网 - 因特网(Internet)
    • 连接在因特网网上的电脑被称为主机(host)
    • internet - 互联网，指由多个计算机网络互联的网络，网络之间的通信协议可以是任意的
    • Internet - 因特网，指全球最大，开发，并由众多网络连接的特定计算机网络，采用TCP/IP协议，前身是美国的ARPANET
  • 发展历程
    • 1969年 - 第一个分组交换网 - ARPANET
    • 1983年 - TCP/IP协议 - 因特网的诞生时间
    • 1985年 - NSFNET - 主干/地区/校园网
    • 1990年 - ARPANET关闭
    • 1991年 - 因特网主干网由私人公司经营
    • 1993年 - NSFNET逐渐被主干网替换，由因特网服务提供者(ISP)运营
    • 1994年 - 万维网WWW技术
    • 1995年 - NSFNET关闭，因特网彻底商业化

# ! ISP(Internet Service Provider)

  • ISP从因特网机构申请成块的IP地址，用户采取购入方式获取IP地址(主机需要IP地址才能进行通信)来接入到因特网

  • 基于ISP的三层结构特征的因特网

    • 相隔较远的主机通信需要经过多个ISP
    • 当用户接入到因特网后也能成为ISP(调制解调器/路由器)

# 因特网

  • 因特网的标准化工作
    • 在制定标准上的特点 - 面向大众
    • RFC技术文档
    • 因特网协会ISOC组织 - 体系结构委员会IAB，工程部IETF，研究部IRTF
  • 因特网的组成
    • 边缘部分 - 用户主机，用于通信，资源共享
    • 核心部分 - 大量网络和路由器，为边缘部分提供服务

# 交换方式

# 电路交换(Circuit Switching)

  • 通过电线将各端的电话两两相连
  • 电话交换机 - 方便通信
    • 每一部电话都连接到交换机上
    • 相当于开关，将需要通信的两部电话接通
  • 电路交换 - 电话交换机接通电话线的方式
  • 电路交换的三个步骤(传输效率低)

# ! 分组交换

  • 报文 - 消息的整块数据，即用户通过主机发送的消息
  • 分组 - 简称为包，指一定长度的数据单元，通常包含数据部分和控制信息
  • 报文发送过程
    • 报文被划分为多个等长数据段
    • 每个数据段的首部(也称包头，包含目的地址)加上控制信息
    • 按照首部目的地址查表转发以寻找合适的接口
    • 通过接口将分组发给下一个分组交换机
    • 将所有分组依次发出，经过途中各分组交换机的存储转发
    • 到达目的地址后，各分组首部被去掉，各数据段组合还原成报文
  • 发送过程的结点的作用
    • 发送方 - 构造分组并发送
    • 路由器 - 缓存分组并转发
    • 接收方 - 接收分组并还原报文

# 计算机网络

# 定义

  • 没有统一明确的定义

  • 最简单的定义 - 一些互相连接并自治的计算机集合

    • 互连 - 有线或无线连接
    • 自治 - 独立的计算机，有软件和硬件，单独允许
    • 集合 - 至少需要两台计算机

  • 较好的定义 - 由一些通用，可编程硬件互连而成

    • 硬件并非专门实现某一特定目的，包括智能手机
    • 可编程硬件可传输多种不同类型的数据，支持多种应用

# ! 分类

  • 按交换技术分类

    • 电路交换网络
    • 报文交换网络
    • 分组交换网络

  • 按使用者分类

    • 公用网(公众网) - 电信公司的大型网络，缴费入网
    • 专用网 - 某部分为单位特殊建造的网络，不向单位以外提供服务

  • 按传输介质分类

    • 有线网络 - 双绞网络，光纤网络等
    • 无线网络 - 无线局域网wifi

  • 按覆盖范围分类

    • 广域网WAN(远程网) - 几十到几千公里的覆盖范围，因特网的核心部分，为核心路由器(跨国的)提供高速连接
    • 城域网MAN - 5到50公里的覆盖范围，作为城市骨干网，互连企业，机构和校园局域网，现代服务的基础设施(电信服务，有线电视服务和互联网服务)
    • 局域网LAN - 1公里的覆盖范围，微型计算机或工作站由高速通信线路(10Mbit/s以上)，单位单独拥有，使用和维护(校园网或企业网)
    • 个域网PAN(个人区域网) - 10m的覆盖范围，用于连接个人使用的设备(键盘，鼠标等)

  • 按拓扑结构分类

    • 总线型网络 - 根传输线将计算机连接，一处故障则全网瘫痪
    • 星型网络 - 每个计算机以单独线路与中央设备(交换机或路由器)连接
    • 环形网络 - 将所有计算机的网络接口连接成一个环(单环和双环，也称令牌环局域网)，信号单项传输
    • 网状网络 - 每个结点至少有两条路径与其他结点相连

# ! 网络的性能指标

# 速率

  • 比特 - 计算机中数据量的单位
    • 8 bit = 1Byte(字节)
    • 1KB =210\small 2^{10}210 B(字节)
    • 1MB = K · KB =220\small 2^{20}220 B
    • 1GB = K · MB =230\small 2^{30}230 B
    • 1TB = K · GB =240\small 2^{40}240 B
  • 速率(比特率/数据率) - 主机在数字信道上传送比特的速率，比特率和数据率
    • 单位 - bit/s, b/s, bps
    • 1kb/s =103\small 10^3103 b/s
    • 1mb/s = 1k · 1kb/s =106\small 10^6106 b/s
    • 1gb/s = 1k · 1mb/s =109\small 10^9109 b/s
    • 1tb/s = 1k · 1gb/s =1012\small 10^{12}1012 b/s

# 带宽(频带宽度)

  • 在模拟信号系统中的意义

    • 表示信号中各成分的各不同频率占据的频率范围
    • 单位 - Hz, kHz, MHz, GHz
    • 300Hz~3.4kHz - 话音主要成分范围

  • 在计算机网络中的意义

    • 表示网络通信线路传输数据的能力
    • 网络带宽 - 单位时间内从网络中的某一点到另一点间能通过的最高数据率
    • 单位 - b/s, kb/s, Mb/s, Gb/s，Tb/s
    • 频带宽度越宽，最高数据率越高

# 吞吐量

  • 吞吐量 - 单位时间内通过某个网络(或信道，接口)的数据量
  • 用于对现实世界网络的测量
  • 受网络的带宽或额定速率限制

# 时延

  • 发送时延 =分组长度(b)发送速率(b/s)\frac{\text{分组长度}(b)}{\text{发送速率}(b/s)}发送速率(b/s)分组长度(b)​，受限于
    • 网卡的发送速率
    • 信道带宽
    • 接口速率
  • 传播时延 =信道长度(m)电磁波传播速率(m/s)\frac{\text{信道长度}(m)}{\text{电磁波传播速率}(m/s)}电磁波传播速率(m/s)信道长度(m)​，电磁波受限场景有
    • 电磁波 - 3 x108\small 10^8108 m/s
    • 铜线中的电磁波 - 2.3 x108\small 10^8108 m/s
    • 光纤中的电磁波 - 2.3 x108\small 10^8108 m/s
  • 处理时延 - 包含排队时延

# 时延带宽积

  • 时延带宽积 = 传播时延 x 带宽
    • 将传输链路看成一个管道，其长度为传播时延，横截面积为带宽，时延带宽积为该管道的体积，管道中充满了比特数据。若发送端连续发送数据，则在发送的第一个比特即将到达终点时，发送端就已经发送了时延带宽积个比特
    • 链路的时延带宽积又称为以比特为单位的链路长度

# 往返时间(Round-Trip Time - RTT)

  • 因特网的信息是双向交互，需要知道双向交互一次所需的时间，即往返时间

# 利用率

  • 信道利用率
    • 表示某信道有百分之几的时间是被利用的(数据通过)
    • 信道利用率增大时，信道引起的时延也会增加，信道利用率不是越高越好
  • 网络利用率 - 全网络的信道利用率的加权平均
  • D网络当前时延D_{\text{网络当前时延}}D网络当前时延​ =D0(网络空闲时的时延)1−U(利用率)\frac{D_0(\text{网络空闲时的时延})}{1-U(\text{利用率})}1−U(利用率)D0​(网络空闲时的时延)​，对应场景有
    • 当网络利用率达50%时，时延加倍
    • 当网络利用率超过50%时，时延剧增
    • 当网络利用率接近100%时，时延趋于无穷大
    • 通常ISP会控制信道利用率不超过50%

# 丢包率

  • 丢包率即分组丢失率 - 一定时间内，传输过程中丢失的分组数量与总分组数量的比率
    • 接口丢包率
    • 结点丢包率
    • 链路丢包率
    • 路径丢包率
    • 网络丢包率
  • 分组丢失的两种情况
    • 传输过程中出现误码，被结点丢弃
    • 分组到达一台队列已满的分组交换机时被丢弃，在通信量较大时就可能造成网络拥塞
  • 丢包率反映的网络拥塞情况
    • 路径丢包率为0 - 无拥塞
    • 路径丢包率为1% ~ 4% - 轻度拥塞
    • 路径丢包率5% ~ 15% - 严重拥塞

# ! 计算机网络体系结构

# 常见网络体系结构

  • OSI体系结构 - 使各种计算机在世界范围内互连成网的标准框架
  • TCP/IP体系结构 - 因特网使用的TCP/IP协议逐步演变成TCP/IP模型
    • 要求用户的主机必须使用TCP/IP协议
    • 不接入因特网也可以使用TCP/IP协议
    • 路由器只包含网络接口层和网际层
OSI 七层模型是什么？每一层的作用是什么？

OSI七层模型是国际标准化组织提出一个网络分层模型，其大体结构以及每一层提供的功能

  • 应用层 - 为计算机用户提供服务
  • 表示层 - 数据处理(编解码、加密解密、压缩解压缩)
  • 会话层 - 管理(建立、维护、重连)应用程序之间的会话
  • 传输层 - 为两台主机进程之间的通信提供通用的数据传输服务
  • 网络层 - 路由和寻址(决定数据在网络的游走路径)
  • 数据链路层 - 帧编码和误差纠正控制
  • 物理层 - 透明地进行比特流传输

每一层都专注做一件事情，并且每一层都需要使用下一层提供的功能，例如传输层需要使用网络层提供的路由和寻址功能，这样传输层才知道把数据传输到哪里去。

OSI七层模型为什么干不过TCP/IP四层模型呢？

有下面几方面原因

  • OSI的专家缺乏实际经验，他们在完成OSI标准时缺乏商业驱动力

  • OSI的协议实现起来过分复杂，而且运行效率很低

  • OSI制定标准的周期太长，因而使得按OSI标准生产的设备无法及时进入市场(20世纪90年代初期，虽然整套的OSI国际标准都已经制定出来，但基于TCP/IP的互联网已经抢先在全球相当大的范围成功运行了)

  • OSI的层次划分不太合理，有些功能在多个层次中重复出现

# TCP/IP体系结构

  • 网络接口层 - 没有具体规定什么内容，是为了互连各不同的网络接口

  • 网际层 - IP协议是该层的核心协议

    • IP协议将不同网络接口互连并提供互连服务给TCP协议和UDP协议

  • 运输层 - TCP和UDP协议是该层的重要协议

    • TCP协议向应用层提供可靠传输服务
    • UDP协议向应用层提供不可靠传输服务

  • 应用层 - 大量应用层协议，如HTTP, SMTP, DNS, RTP协议等

  • TCP协议和IP协议是非常重要的两个协议，并用这两个协议表示整个协议家族，常称为TCP/IP协议族

    • 嵌入式开发领域中，TCP/IP协议族常被称为TCP/IP协议栈(分层结构与栈结构类似)

  • 在学习计算机网络原理时会采取的折中方法 - 综合OSI体系结构和TCP/IP协议的优点，采用五层协议体系

# 分层的必要性

计算机的系统非常复杂，分层可将其转化为若干较小的局部问题，各层对应的问题有

  • 物理层
    • 采用什么样的传输介质？
    • 采用什么样的物理接口？
    • 采用什么样的信号表示比特0和1？(并非方波信号)
  • 数据链路层
    • 如何识别网络中的各主机(主机编址问题，MAC地址)？
    • 如何从信号所表示的一连串比特流中区分地址和数据？
    • 如何协调各主机争用主线？
  • 网络层
    • 如何标识各网络以及网络中的各主机(网络和主机共同编址问题，IP地址)？
    • 路由器如何转发分组，如何进行路由选择？
  • 运输层
    • 如何解决进程之间基于网络的通信问题？
    • 出现传输错误时，如何处理？
  • 应用层
    • 通过应用进程间的交互来完成特定的网络应用
    • 支持万维网应用的HTTP协议
    • 支持电子邮箱的SMTP协议
    • 支持文件传送的FTP协议

  • 分层思想的演示
为什么网络要分层？(从后台程序上看)

复杂的系统需要分层，因为每一层都需要专注于一类事情。网络分层的原因也是一样，每一层只专注于做一类事情。主要有3方面的好处

  1. 各层之间相互独立 - 各层之间相互独立，各层之间不需要关心其他层是如何实现的，只需要知道自己如何调用下层提供好的功能就可以了(可以简单理解为接口调用)，这与开发时系统进行分层是一个道理
  2. 提高了灵活性和可替换性 - 每一层都可以使用最适合的技术来实现，只需要保证提供的功能以及暴露的接口规则没有改变就行了。并且，每一层都可以根据需要进行修改或替换，并且不会影响到整个网络的结构。这个与平时开发系统时要求的高内聚、低耦合的原则也是可以对应上的
  3. 大问题化小 - 分层可以将复杂的网络问题分解为许多比较小的、界线比较清晰简单的小问题来处理和解决。这样使得复杂的计算机网络系统变得易于设计，实现和标准化。 这个与平时开发时将系统功能分解，然后将复杂的问题分解为容易理解的更小的问题是相对应的，这些较小的问题具有更好的边界(目标和接口)定义

# 协议

  • 协议 - 控制两个对等实体进行逻辑通信的规则的集合
  • 协议三要素
    • 语法 - 定义所交换信息的格式 - IP数据报，TCP报文段，HTTP报文
    • 语义 - 定义收发双方所需要完成的操作
    • 同步 - 定义收发双方的时序关系(不是指时钟频率)
