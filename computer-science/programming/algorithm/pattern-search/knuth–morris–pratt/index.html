# KMP

# 实质

KMP算法是一种用于字符串匹配的算法，其主要目的是在一个文本串中查找一个模式串的出现位置。实质是通过预处理模式串，构建一个部分匹配表(Partial Match Table - PMT)，然后利用这个表在文本串中快速定位可能发生不匹配的位置，从而减少不必要的比较次数。

KMP算法的主要思想是，当在文本串中发现不匹配的字符时，利用已经匹配的部分信息，尽量减少模式串与文本串的比较次数，直接跳过一些不可能匹配的位置。

KMP算法的实质步骤如下

  1. 构建部分匹配表(Partial Match Table) - 对模式串进行预处理，生成一个部分匹配表，该表记录了模式串中每个位置对应的最长前缀和最长后缀的匹配长度
  2. 利用部分匹配表进行匹配 - 在匹配过程中，当发生不匹配时，根据部分匹配表中的信息，移动模式串的相对位置，而无需回溯文本串

KMP算法的优点在于它可以在最坏情况下达到线性时间复杂度，即O(m + n)，其中m是模式串的长度，n是文本串的长度。这使得KMP算法在处理大规模文本数据时具有高效性能。

# 核心

Prefix table(前缀表) / LPS table(Longest Prefix Suffix)(最长公共前后缀表)

# prefix / suffix

例如

"ababcb"

a => 0

ab => a | b => 0

aba => a ab | ba a => 1

abab => a ab aba | bab ab b => 2

ababc => a ab aba abab | babc abc bc c => 0

ababcb => a ab aba abab ababc | babcb abcb bcb cb b => 0
