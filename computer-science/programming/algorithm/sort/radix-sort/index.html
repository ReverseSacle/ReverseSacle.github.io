# Radix Sort

基数排序是一种借助多关键字排序思想对单逻辑关键字进行排序的方法，与其他排序方法不同的是该排序中没有数据的比较和记录移动的操作。其中关键字排序的实现通常有两种方法，一是最高位优先(Most Significant Digit first - MSD)；二是最低位优先(Least Significant Digit first - LSD)。两种方法的约束中只规定了按照什么次序进行排序，但未规定排序时使用的方法。此处采用了LSD方法，并通过分配(Distribute)与收集(Collect)来实现排序。

基数排序适用于整数或字符串的排序。

 

基数的选取通常有两种，一是数值范围在[0, 9]内，基数选取值为10；二是字母范围在[1,26]，基数选取值为26。

由于待排序的数值不超十位时，因此只需两次分配与收集即可，分配时以数值位数(十位与个位)来分配，收集时，以基数辅助空间顺序收集。

Radix Sort

时间复杂度：O(d(n+rd)d(n+rd)d(n+rd))

空间复杂度：O(rdrdrd)

 

由于基数空间位置的选取是依靠数值位数决定，但当数值中存在负数时，由于负数的位置不存在便产生了错误。为了解决该错误，可通过将基数空间提升至20，下标[1,9]的空间中存储[-9, -1]的值，而下标[10,19]则存储正整数。负数的值反向存储是因为，负数值越大越小。
