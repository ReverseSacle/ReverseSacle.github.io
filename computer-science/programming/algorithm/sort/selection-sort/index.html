# 简单选择排序(Simple Selection Sort)

是一种比较直觉的排序算法，从头开始依次选择最小值，每当选择完一个最小值，选择范围便会从前面缩小一格。

其实现方法较为简单，设nin_ini​，nin_ini​的值在[nin_ini​,nnn]区间内选择，选择区间内最小值放入。

Simple Selection Sort

时间复杂度：O(n2n^2n2)

空间复杂度：O(1)

 

# 树形选择排序(Tree Selection Sort)

又称锦标赛排序(Tournament Sort)，一种对简单选择排序的改进方案，基于锦标赛的选择排序思想，通过完全二叉树来实现。

由于简单选择排序中有大量的比较次数，其通过完全二叉树结构将真实值比较结果存入在根结点中，根结点的比较结果存入在根的父结点中，以此往复，将所有真实值都进行了一次比较和存储。

其实现方法有点复杂，但实现原理比较简单。

首先建立排序需要的完全二叉树，其所需结点个数为2n−1\small 2n - 12n−1个。

带下标

不带下标

 

再进行初始化(第一次排序)，只有叶子结点存储真实值，其他根结点都存储真实值下标。

倘若当前根结点的左右结点为叶子结点时，其存储的值为下标；

倘若当前根结点的左右结点为根结点时，通过其左右结点根所存储的下标值索引到真实值后再进行比较，比较完后其存储的值为其左右结点所存储的下标。

由于先进行了一次初始化(第一次排序)，预先存储了所有真实值两两比较的结果在它们的父结点中，其头根结点便存储了所有真实值比较后的最小值下标，这也是第一次排序后得出的第一个最小值。由于比较过程有缓存，之后的排序中，已不需要再全部两两比较了。由于排序后头根会存储比较后最小值的下标，因此之后的排序会从由头根保存的最小值下标位置开始并沿着其父结点往上爬。

Tree Selection Sort

时间复杂度：O(nlognnlog nnlogn)

空间复杂度：O(mmm)

 

# ! 堆选择排序(Heap Selection Sort)

基于树形选择排序的优化排序方法，通过对堆作出定义，并以此约束对数组进行排序，此过程可用一棵有nnn​个叶子结点的完全二叉树表示。

堆选择排序基于二叉堆数据结构，利用了堆的性质来实现排序。堆是一个特殊的树状数据结构，其中每个节点的值都大于或等于其子节点的值(最大堆)，或者小于或等于其子节点的值(最小堆)。

堆定义

由于树形选择排序使用了大量的额外空间，其通过堆定义的约束减少了排序所需的额外空间。

其实现方法较为复杂，首先将整个数组抽象为一棵完全二叉树。

带下标

不带下标

 

再进行第一次排序(小堆)，从倒数第一个根节点开始至头结点比较其左右孩子结点，其左右孩子结点比较得出的最小值会与它们父根结点的值再进行比较并以值交换的方式存储最小值。

 

之后的排序过程类似树形选择排序，

倘若采用了小堆的约束，由于第一次的排序，头根已存储所有的值两两比较的最小值，将其与(n−in-in−i)数组的最后一个值交换后再进行之后的排序。之后的排序中，每次从头根开始沿着最小左或右孩子结点往下延伸，每次排序完(n−in-in−i)数组的最小值都会存储在头根，再将头根值与(n−in-in−i)数组的最后一个值交换，以此往复最小值逐趟存储在(n−in-in−i)数组的最后一位，以此完成排序；

倘若采用了大堆的约束，与小堆约束排序不同的是每次排序完后，数组的最大值会逐趟存储在(n−in-in−i)数组的最后一位并以此完成排序。

Heap Selection Sort

时间复杂度：O(nlognnlog nnlogn)

空间复杂度：O(1)
