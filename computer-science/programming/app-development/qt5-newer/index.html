# 安装

→官网下载地址, Qt6, 滑动到最后面, 点击Qt Online Installer - 开源版

在Qt 6中，官方不再提供对32位系统的官方支持，对win7及以下不支持了。不过，Qt 5仍然支持32位Windows平台。

→官网下载地址, Qt5, 5.14.2版本, 最后一个可离线下载的版本 - 断开网络后安装

MSVC编译需要安装对应的Visual Studio软件。例如，MSVC 2015是搭配VS 2015(版本14.0)，当系统没有安装相应的Visual Studio或Visual Studio未配置C++时，使用MSVC 2015是无法运行的。

# MSVC配置(visual studio 2022, 即高版本VS的配置)

  • 安装Visual Studio 2022，只选择使用C++的桌面开发，右边选择栏选择

    • Qt6选 - MSVC v143 - VS 2019 C++ x64/x86 生成工具

    • Qt5选 - MSVC v142 - VS 2017 C++ x64/x86 生成工具

    • 安装完不用打开

  • windows通过左下搜索，搜索应用，选择应用和功能，进入后里面搜索栏处搜索software，找到Windows Software Development Kit，选择修改。进入后，选择change → Debugging Tools for Windows，确认即可

  • 打开Qt Creator，顶栏Tools → Options → Kits → Compilers → 选择Manual中的Add → MSVC → C++, Initialization中填入(根据自己的VS路径填写)D:\software\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvarsall.bat。32位项目选x86，64位项目选x86_amd64，之后的ABI选择<custom>以此配置为x86-windows-msvc2017-pe-64bit，配置好后点右下角的Apply。之后再在Manual中的C++一栏选中你刚刚配置好的，并在右边栏中点击Clone，再将ABI换为32-bit

  • 接着返回到Kits，依次选择Desktop Qt %{Qt:Version} MSVC2017 32bit与Desktop Qt %{Qt:Version} MSVC2017 64bit，两者的Compiler中C一栏都不填，C++一栏选择刚刚创建的对应32bit/64bit，Debugger一栏选对应32位/64位的cdb.exe

  • 接着，顶栏Help → About Plugins → 搜索Clang，取消勾选ClangCodeModel，重启Qt creator

  • Qt 5中MSVC与CMake搭配的项目会一直报错，兼容性问题，目前没有什么好方法

  • 如果创建新项目提示缺少cstddef.h，可用windows左下搜索栏，搜索visual studio install，进入后点击更多中的修复，重启后即可解决

# 添加QSS样式

Qt Creator中，选中项目根目录文件夹，右键 → Add new... → Qt → Qt Resource File → 写个名字，不用添加后缀 → 添加完后，中间板块，点击Add Prefix → 底下Prefix一栏改为/ → 点击Add Files，添加.qss后缀的文件

# 关于QWebEngineView编译

需要120多G的剩余存储，确保安装时选了源码和QWebEngineView，大概六个多小时，最快三个多小时。

python2.7.15 - 通过搜索2.7.15，下载MSI文件

Perl(Strawberry)

Bison and flex - 下载zip文件，解压后将win-bison.exe改为bison.exe, win-flex.exe改为flex.exe`，并将该根目录添加到系统环境

Gperf - 选Complete package, except sources那个

确保这几个都在系统环境中。

VS2017编译工具

  • 目录(根据安装位置寻得) - D:\software\Program Files\Microsoft Visual Studio\2017\Community\VC\Auxiliary\Build
  • 编译32位选 - vcvars32.bat
  • 编译64位选 - vcvars64.bat

nmake路径

  • 编译32位选 - D:\software\Program Files\Microsoft Visual Studio\2017\Community\VC\Tools\MSVC\14.14.26428\bin\Hostx86\x86\nmake.exe
  • 编译64位选 - D:\software\Program Files\Microsoft Visual Studio\2017\Community\VC\Tools\MSVC\14.14.26428\bin\Hostx64\x64\nmake.exe

Qt的QtWebEngine目录 - D:\software\Qt5.14.2\5.14.2\Src\qtwebengine

Qt的qmake目录

  • 编译32位选 - D:\software\Qt5.14.2\5.14.2\msvc2017\bin\qmake.exe
  • 编译64位选 - D:\software\Qt5.14.2\5.14.2\msvc2017_64\bin\qmake.exe

Windows PowerShell中 - 32位编译

  • cd “D:\software\Program Files\Microsoft Visual Studio\2017\Community\VC\Auxiliary\Build”
  • ./vcvars32.bat
  • cd "D:\software\Qt5.14.2\5.14.2\Src\qtwebengine"
  • D:\software\Qt5.14.2\5.14.2\msvc2017\bin\qmake.exe -- -webengine-proprietary-codecs
  • 没有类似WARNING: Qt WebEngine on Windows requires a Windows SDK version 10.0.18362 or newer.即可进行下一步，只需注意WARNING写的东西即可
  • D:\software\Program Files\Microsoft Visual Studio\2017\Community\VC\Tools\MSVC\14.14.26428\bin\Hostx86\x86\nmake.exe
  • 编译完执行，D:\software\Program Files\Microsoft Visual Studio\2017\Community\VC\Tools\MSVC\14.14.26428\bin\Hostx86\x86\nmake.exe install

# 打包对象

release模式运行后~/(项目目录)/(项目名)/build-untitled-Desktop_Qt_5_14_2_MSVC2017_32bit-Release\release目录下会生成(项目名).exe文件。

# 工具

windeployqt - Qt官方开发环境里自带的一个工具，

  • 主要用于Qt生成的.exe文件的依赖获取。它会将(.exe)程序用到的所有Qt库文件拷贝到(.exe)程序所在的文件夹中

  • 执行完后，还不能在没有安装Qt的电脑上运行，还需将

    • 目录 - D:\software\Qt5.14.2\5.14.2\mingw73_32\bin
    • mingw73_32为当前项目使用的编译环境，依据自己的项目而定
    • 复杂程序中可能还有一些图片文件、数据库文件、配置文件之类的，需要自己复制这些文件到打包好的文件夹里
    • 用虚拟机试了一下，使用windeployqt打包，即使有第三方库依旧可以，不需要手动将第三方库复制到(.exe)程序所在的文件夹中

  • (注意)windeployqt不适用于部署用C/C++编写的原生可执行文件

    • 当用于C/C++编写的原生可执行文件的打包时，会报错xxx.exe does not seem to be a Qt executable.
    • 原生可执行文件只需libgcc_s_dw2-1.dll, libstdc++-6.dll, libwinpthread-1.dll这几个文件即可在其他电脑上运行，不需要Qt的依赖
    • 如果打包后有错误，可以尝试的解决方法
      • (所需文件)~/(项目目录)/(项目名)/build-untitled-Desktop_Qt_5_14_2_MSVC2017_32bit-Release\release目录下的libgcc_s_dw2-1.dll, libstdc++-6.dll, libwinpthread-1.dll这几个文件拷贝到(.exe)程序所在的文件夹中
      • 保险起见，可以把该目录下所有.dll文件都拷贝走

# 打包过程

(打包前需知)注意对应你的编译环境，这里是MSVC2017_32bit，看清文件夹的编译环境。

(可选一)添加应用程序图标

  • 将.ico文件放大项目的根目录
  • .pro文件中添加RC_ICONS = 你的文件名.ico

(可选二)添加应用程序版本号

  • .pro文件中添加VERSION = 1.0.0

打开Qt Creator，左下工具栏，点击第一个电脑图标，在右边选Release，接着点击该图标的下面的图标，build好之后，关闭打开的程序。

新建一个文件夹，将~/(项目目录)/(项目名)/build-untitled-Desktop_Qt_5_14_2_MSVC2017_32bit-Release\release目录下的.exe文件拷贝到该文件夹中。

windows左下搜索栏搜索Qt，点击Qt 5.14.2 (MSVC 2017 32-bit)
