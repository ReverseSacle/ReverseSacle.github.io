# 内联函数(inline)

语法 - 在函数定义前面加上关键字inline，倘若有函数声明，其前面也要加上关键字

inline关键字的作用是建议编译器将函数调用替换为函数体，以减少函数调用的开销。使用inline函数的目的是提高程序的运行效率，特别是那些频繁调用的小函数。

内联函数在编译阶段展开，编译器在编译过程中会将函数的内容直接嵌入到函数调用的地方，不能进行递归操作。

内联函数能节省时间，但消耗内存，如果函数体过大，编译器可能不将其作为内联函数。

编译器对inline函数的处理步骤？

  • 函数体内联 - 当编译器遇到一个inline函数的调用时，它可以选择性地在调用点将函数体直接插入到调用位置，而不是生成一个函数调用。这样可以减少函数调用的开销，因为没有了函数调用和返回的额外开销

  • 函数体复制 - 如果编译器决定内联函数，它会在每个调用点将函数体的副本插入代码中。这意味着在程序中可能会有多个相同的函数体的副本，这会增加代码的量

  • 优化 - 内联函数的内联展开可能导致更多的优化机会。由于函数体插入到调用点，编译器可以更好地进行局部优化，例如在内联的上下文中进行常量折叠和其他优化

  • 链接 - 如果多个源文件包含相同的inline函数定义，编译器通常会选择其中一个进行内联，并在链接时合并多个定义，以避免重复定义错误

inline关键字是对编译器的建议，而不是强制。编译器可以忽略inline关键字，并根据自身的优化策略来决定是否内联某个函数。

宏定义和展开、内联函数区别？

内联函数是在被调用的地方直接插入代码，而不是通过标准的函数调用机制。这就意味着，当程序执行时，不再有真正的函数调用和返回过程，而是在调用点直接执行内联函数的代码，这有减少了函数调用的开销。在过程化集成方面，通过将内联函数的代码插入到调用点，编译器可以更好地进行上下文优化，因为它可以看到调用点周围的代码，这有助于提高代码的执行效率。

宏定义是在预处理阶段进行文本替换，不会对传递给它的参数进行任何类型检查。而，内联函数在编译阶段会进行类型检查，在调用时会检查传递给它的参数的数据类型，以及返回值的数据类型，因此在参数类型检查上更加安全。内联函数与宏类似，而区别在于，宏是由预处理器对宏进行替代，而内联函数是通过编译器的控制来实现的，并且内联函数是真正的函数，只是在需要用到的时候，内联函数才会像宏一样展开，这样一来就取消了函数的参数压栈，减少了调用的开销。

此外，宏是预编译器的输入，宏展开之后的结果会送去编译器做语法分析。宏与函数处于不同的级别，操作不同的实体。宏操作的是token, 可以进行token的替换和连接等操作，在语法分析之前起作用。而函数是语言中的概念，会在语法树中创建对应的实体，内联只是函数的一个属性。

有了函数，还要宏和内联函数有什么用？

  1. 函数并不能完全替代宏，有些宏可以在当前作用域生成一些变量，但函数做不到
  2. 内联函数只是函数的一种形式，内联是给编译器的提示，告诉它最好把这个函数在被调用处展开，省掉一个函数调用的开销(压栈，跳转，返回)

内联函数也有一定的局限性，就是函数中的执行代码不能太多。如果，内联函数的函数体过大，一般的编译器会放弃内联的方式，而采用普通的方式调用函数，此时内联函数的执行效率就和普通函数一样了。

什么时候该使用内联函数？

  • 小型函数 - 内联函数适用于函数体较小的函数。由于内联函数会直接展开代码，避免了函数调用的开销，对于函数体较小的情况，内联可以提高执行效率

  • 频繁调用的函数 - 内联函数适用于在程序中频繁调用的函数。由于内联函数减少了函数调用的开销，对于频繁调用的函数，内联可以提高程序的整体性能

  • 短小的循环体 - 内联函数适用于循环体较短小的情况。循环内的函数调用会在每次迭代时增加开销，使用内联函数可以减少这种开销

  • 在类定义中的成员函数(C++) - 类定义中的成员函数通常可以声明为内联函数。这样做可以避免函数调用的开销，对于一些简单的成员函数，内联是一个合理的选择

  • 避免代码重复 - 内联函数可以避免由于多次调用而导致的代码重复，提高代码的可维护性和可读性
