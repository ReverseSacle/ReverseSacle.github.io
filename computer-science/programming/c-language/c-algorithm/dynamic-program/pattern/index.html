# Pattern

LC.509 Fibonacci Number

Fibonacci Number

第0个斐波那契数f0=0f_0 = 0f0​=0，第1个斐波那契数f1=1f_1 = 1f1​=1，第2个斐波那契数，

f2=f0+f1=1f_2 = f_0 + f_1 = 1f2​=f0​+f1​=1，当前状态中f0=0, f1=1f_0 = 0 \text{, }f_1 = 1f0​=0, f1​=1，当求第3个斐波那契数时，状态将会被更新，

此时状态将呈现交替形式，f0=f1=1, f1=f2=1f_0 = f_1 = 1 \text{, }f_1 = f_2 = 1f0​=f1​=1, f1​=f2​=1，即f1=1, f2=1f_1 = 1 \text{, }f_2 = 1f1​=1, f2​=1，故

f3=f1+f2=1f_3 = f_1 + f_2 = 1f3​=f1​+f2​=1，最终呈现的过程fn=fn−1+fn−2f_n = f_{n-1} + f_{n-2}fn​=fn−1​+fn−2​，如题目所描述。

由于编程中变量只初始化一次，值得交替如上述所表达的那样，故此过程就相当于状态的转移。

LC.70 Climbing Stairs

Climbing Stairs

爬第一层，只可以从第零层爬一层，即f0=1f_0 = 1f0​=1，

爬第二层，可以从第一层爬一层(f1=1f_1 = 1f1​=1)或从第零层爬两层(f0=1f_0 = 1f0​=1)，即f2=f0+f1=2f_2 = f_0 + f_1 = 2f2​=f0​+f1​=2，

爬第三层，可以从第二层爬一层(f2=2f_2 = 2f2​=2)或从第一层爬一层(f1=1f_1 = 1f1​=1)，即f3=f1+f2=3f_3 = f_1 + f_2 = 3f3​=f1​+f2​=3，

由此可见，每次爬当前层时，都需要从下一层或下二层开始爬，并且下一层与下二层又会从下下一层或下下两层爬，也就是说爬到当前层是经历了爬之前楼层的状态，并且状态是逐层发生转移的。

这里的状态是什么意思？假如你在走楼梯，你一次走了一层，你的状态就从第零层转移到第一层了，你接着一次走了两层，此时你的状态就从第一层转移到第三层了。此时，当前的你在第三层这个状态是，你从第零层走了一层到第一层与从第一层走了两层到第三层，这两个过程的状态的叠加而成的结果。

接着，将上面的例子用到此题上，爬到当前层的状态等于爬到下一层的状态加上下两层的状态叠加，

爬第一层时，有一种方法，即只可以从底部爬一层，第零层的状态为0表示初始状态；

爬第二层时，有两种方法，即可以从第一层爬一层或从第零层爬两层，第零的状态为0，第一层的状态为1；

爬第三层时，有三种方法，即可以从第二层爬一层或从第一层爬两层，第一的状态为1，第二层的状态为2；

由于变量名不可变，层数状态的转移过程相当于 - 例如，f0=f1=1, f1=f2=2, f3=f1+f2=3f_0 = f_1 = 1 \text{, }f_1 = f_2 = 2 \text{, } f_3 = f_1 + f_2 = 3f0​=f1​=1, f1​=f2​=2, f3​=f1​+f2​=3(第三层)。

LC.746 Min Cost Climbing Stairs

Min Cost Climbing Stairs

带初始状态和状态转移条件的爬楼梯。题目要求的top of the floor即数组最后一位的后一位。

只需按照LC.70 Climbing Stairs的方法，

将初始状态设置好(f0=cost[0], f1=cost[1]f_0 = cost[0] \text{, }f_1 = cost[1]f0​=cost[0], f1​=cost[1])，再将状态转移过程变更为(f2=min(f0,f1)+cost[i]f_2= min(f_0,f_1) + cost[i]f2​=min(f0​,f1​)+cost[i])即可。

LC.91 Decode Ways

Decode Ways

由于约束的存在，'0'字符不能编码。此外，除了第1个字符位，只能编码其当前位外，其他字符位，要编码当前位非零字符，要么编码[10,26]范围内的两个字符组合。

倘若编码当前编码位(字符位大于1)，要从前一个编码位编码当前非零字符位，要么从前两个编码位编码[10,26]范围内的两个字符组合。

也就是说，编码当前位(字符位大于1) = (零 or 从前一个编码位编码当前位) + (零 or 从前两个编码位编码[10,26]范围内的两个字符组合)
