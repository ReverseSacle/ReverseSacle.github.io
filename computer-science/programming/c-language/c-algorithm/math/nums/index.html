首先，所有的丑数都是与2、3或5的乘积得来的，并且乘数必然是丑数。(可通过一一列举验证)

倘若，知道了1为丑数，那下一个丑数必然是1 * 2 或 1 * 3 或 1 * 5中的一个，显然，候选的丑数有三个，但下个丑数只能是一个，但选择了一个丑数后，另外两个丑数又该怎样呢？要是将候选丑数全部加入到已求得的丑数当中，那么之后像2 * 3遇上3 * 2这样的情况，会额外增加计算空间，再将之前已求得的丑数再求取一遍；要是，我们保留未被选上的候选丑数的候选状态并更新被选上的候选丑数的候选状态，当我们求得下个丑数为2时，由于我们保留了未被选上的候选丑数的候选状态，此时，下一个丑数会从2 * 2或1 * 3或1 * 5中选取，显然通过这种方法选取丑数并不会出现额外计算的情况。那么，从当前位置选取丑数时下一个丑数一定是从x * 2 或 y * 3 或z * 5中选取，x、y和z则表示丑数选取的候选状态。

LC.470 Implement Rand10() Using Rand7()

假如有6个骰子，倘若想取得六个骰子之和的最大数(tail)与最小数(head)，可以投掷这些骰子，如果六个之和为1则取得了最小数(head)，如何还想取得其他数，只需投掷骰子即可。

假设已经有了rand10()迭代器，每次能取得0~9之间的一位数，如果想通过rand10()实现rand100()/rand1000()，

rand100() = 10 * rand10() + 1 * rand10()

其范围为10 *（0 ~ 9）+ 1 * （0 ~ 9）= 0 ~ 99

rand1000() = 100 * rand10() + 1 * rand10()

其范围为100 * （0 ~ 9）+ 1 * （0 ~ 9）= 0 ~ 999

相似地，假设已经有了rand5()迭代器，想实现rand25()/rand125()

rand25() = 5 * rand5() + 1 * rand5()

其范围为5 * (0 ~ 4) + 1 * (0 ~ 4) = 0 ~24

rand125() = 25 * rand5() + 5 * rand5() + 1 * rand5()

其范围为25 * （0 ~ 4) + 5 * (0 ~ 4) + 1 * (0 ~ 4) = 0 ~ 124

但，要是rand10()，每次能取得1~10之间的一位数，又要如何实现rand100()，

rand100() = 10 * (rand10() - 1) + 1 * (rand10() - 1)

其范围为10 * （(1 ~ 10) - 1) + 1 * ((1 ~ 10) - 1) = 0 ~ 99

依据以上规则，如今已有rand7()，每次能取得1 ~ 7之间的一位数，想实现rand10()并且其每次能取得1~10之间的一位数

rand49() = 7 * (rand7() - 1) + 1 * (rand7() - 1)

其范围为7 * （(1 ~ 7) - 1) + 1 * （(1 ~ 7) - 1) = 0 ~ 48

rand10() = rand40() % 10 + 1

其范围为 (0 ~ 39) % 10 + 1 = 1 ~ 10

由于rand7()无法通过扩展来达到(0 ~ 39)的范围，因此通过限制rand49()中(40 ~ 48)该范围，让其固定获取(0 ~ 39)之间的一位数，以此达到了rand40()的（0 ~ 39）的获取范围。
