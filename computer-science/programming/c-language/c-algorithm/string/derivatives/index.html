基于LC.12 Integer to Roman

阿拉伯数字转中文数字

假如有以下阿拉伯数字转成中文

千分位以内

1 => 一

10 => 十

11 => 十一

101 => 一百零一

1001 => 一千零一

1010 => 一千零一十

1100 => 一千一百

万分位以内

11000 => 一万一千

10100 => 一万零一百

10010 => 一万零一十

10001 => 一万零一

10000 => 一万

110000 => 十一万

101000 => 十万零一千

100100 => 十万零一百

100010 => 十万零一十

100001 => 十万零一

——————

万位与千位切分

11000 =>

1 => 一万

1000 => 一千

——

10100 =>

1 => 一万零

100 => 一百

——

110000 =>

11 => 十一万

0000 => 无

由于亿位与万位的区别不大，此处忽略了。

接下来看看下面的数字

60090 => 六万零九十

60900 => 六万零九百

69000 => 六万九百

小结下，

|1亿2千万3百万4十万5万6千7百8十9个\underset {\text{亿}}{1} \underset {\text{千万}}{2} \underset {\text{百万}}{3} \underset {\text{十万}}{4} \underset {\text{万}}{5} \underset {\text{千}}{6} \underset {\text{百}}{7} \underset {\text{十}}{8} \underset {\text{个}}{9}亿1​千万2​百万3​十万4​万5​千6​百7​十8​个9​ |

由上面的例子来表述，

中文的数字表示基础为千分位以内，上面的数字可以分区域表示，

亿

1 => 一亿

万

2345 => 两千三百四十五万

千

6789 => 六千七百八十九

总体就表示为一亿两千三百四十五万六千七百八十九

除此外还有两个约束，一是十该如何表示，二是什么时候补零？

由上面的千分位以内的例子可知，当有一十存在并且没有更高位表示时，一十表示为十；

由上面的所有例子可知，当前位表示与下一位表示中，倘若有一个缺位存在时补一个零，

假如当前高位为10的整数并且还存在下一位，此时该高位后面必然需补一个零；

假如当前位存在多个空缺零，比如1010，其也满足规则，其表示为一千零一十，千位与十万中间缺了百位。
