# 二叉树(Binary Tree)

树是n个结点的有限集。在任意一棵非空树中

  • 只有一个被称为根的结点

  • 当结点大于1时，其余结点又可分为m个互不相交的有限集，每一个有限集又是一棵树并被称为根的子树

二叉树

二叉树是一种树状结构，子树有左右之分，其中每个节点最多有两个子节点，分别称为左子节点和右子节点，次序不能任意颠倒。

基本特性

  • 根节点(Root) - 二叉树的顶部节点，没有父节点
  • 叶子节点(Leaf) - 没有子节点的节点称为叶子节点
  • 内部节点 - 除了叶子节点，其他节点都是内部节点
  • 子树 - 以某个节点为根节点的子树，由该节点的左子节点和右子节点以及它们的后代组成
  • 深度(Depth) - 从根节点到某个节点的路径的长度，根节点的深度为0
  • 高度(Height) - 从某个节点到它的最远叶子节点的路径的长度，叶子节点的高度为0
  • 度数 - 节点拥有的子节点的数量，二叉树的度数最大为2

满二叉树(Full Binary Tree)

所有非叶子节点都有两个子节点，每一层都被完全填充

  • 深度为k的满二叉树必然包含2k−1\small 2^k-12k−1个节点
  • 包含n个节点的满二叉树的深度为log2(n+1)\small log_2(n+1)log2​(n+1)
完全二叉树(Complete Binary Tree)

除了最后一层，其他层都被完全填充，且最后一层的节点都靠左排列。

包含nnn个节点的完全二叉树的深度为floor(log2n)+1\small floor(log_2n)+1floor(log2​n)+1

二叉排序(搜索/查找)树(Binary Search Tree - BST)

二叉排序(搜索)树有可能是一颗空树，或者是具有以下性质的二叉树

  1. 其左、右子树也分别为二叉树
  2. 当左子树不空时，其左子树上所有结点的值均小于其根结点的值
  3. 当右子树不空时，其右子树上所有结点的值均大于其根结点的值

平衡二叉树(AVL Tree)

平衡二叉树，基底为二叉搜索树，它可能是一颗空树，或者是具有以下性质的二叉树

  1. 左右子树都是平衡二叉树

  2. 左右子树的深度差绝对值不超过1

平衡二叉树上的每个结点都有一个平衡因子BF(Balance Factor)，通常定义为该结点的左子树的深度减去右子树的深度(实际上更建议用右子树减去左子树)，因此平衡二叉树上所有结点的平衡因子只可能是-1,0,1。当存在二叉树上一个结点的平衡因子的绝对值大于1的情况，则此二叉树就为不平衡状态。

线索二叉树/线索链表(Threaded BinaryTree)

为了得到遍历时结点在任一序列中的前驱与后继信息，通过二叉链表中存在的n+1各空链域来定义一个能存放前驱与后继信息的结点。倘若结点有左子树，左孩子结点用l_child来指示，否则指示其前驱；倘若结点有右子树，右孩子结点用r_child来指示，否则指示其后继。

为了避免混淆，增加了两个标志域L_Tag与R_Tag，结构为l_child | L_Tag | data | R_Tag | r_child，以这种结点构成的二叉链表作为二叉树的存储结构，叫做线索链表，其中指向结点前驱与后继信息的指针，叫做线索。对于二叉树，以某种次序遍历使其变为线索二叉树的过程叫做线索化。

红黑树(Red Black Tree)

红黑树是一种自平衡二叉搜索树，它在每个节点上增加了额外的信息，以确保树在进行插入和删除等操作时保持平衡。

红黑树的特点

  • 每个节点被标记为红色或黑色
  • 根节点是黑色的
  • 叶子节点(NIL或空节点)被认为是黑色的
  • 不能有两个相邻的红色节点，即红色节点不能相连
  • 从根节点到每个叶子节点的路径上，黑色节点的数量是相同的

应用场景

  • 数据库系统 - 许多数据库系统使用红黑树来实现索引结构，如在B+树的实现中。这提供了高效的查找和范围查询性能。

  • C++ STL中的std::map和std::set - 在C++标准模板库(STL)中，std::map和std::set通常使用红黑树作为底层数据结构，以实现有序的关联容器

  • Linux内核中的虚拟内存管理 - 在Linux内核中，红黑树被广泛应用于虚拟内存管理，用于高效地管理进程地址空间

  • 计算机网络中的路由表 - 路由表的实现通常使用红黑树，以便快速查找和更新路由信息

  • 文件系统 - 一些文件系统中的目录结构可以使用红黑树来加速文件的查找和访问

哈夫曼树(Huffman Tree)与编码

哈夫曼树是一种二叉树，被用于数据压缩中的哈夫曼编码(Huffman Coding)算法。该算法是由大卫·哈夫曼(David A. Huffman)于1952年提出的，主要用于实现数据的有损压缩，即通过对数据进行编码和解码来减少其表示所需的位数。

哈夫曼树的构建过程

  • 权重赋值 - 对于给定的一组数据，首先计算每个数据的权重(频率)

  • 构建最小堆 - 将每个数据看作一个节点，构建一个最小堆(或最小优先队列)，按照节点的权重作为关键字进行排列

  • 合并节点 - 从最小堆中取出两个权重最小的节点，合并为一个新节点，其权重为两个节点的权重之和。将这个新节点重新插入最小堆

  • 重复 - 重复上述合并过程，直到最小堆中只剩下一个节点，这个节点就是哈夫曼树的根节点

哈夫曼编码

哈夫曼编码是一种变长编码，它根据数据的权重来分配不同长度的二进制码。权重越高的数据，对应的编码越短，权重越低的数据，对应的编码越长。这种编码方式保证了没有其他编码能够表示的二进制串的前缀等于任何权重较大的数据的编码。

应用场景

  • 数据压缩 - 主要应用于数据压缩领域，例如在图像、音频和视频文件的压缩中。通过使用哈夫曼编码，频繁出现的数据可以用较短的编码表示，从而实现对数据的高效压缩

  • 通信 - 在通信领域，哈夫曼编码常用于数据传输。通过使用变长编码，可以减少需要传输的比特数，提高通信效率

  • 存储 - 在存储系统中，哈夫曼编码也被用于提高存储效率。通过使用较短的编码表示频繁出现的数据，可以减小存储空间的需求

  • 编解码器设计 - 哈夫曼编码常被用于设计编解码器，例如在音频和视频编码标准中，如MP3和H.264

  • 字典压缩 - 在一些字典压缩算法中，哈夫曼编码用于构建字典，以实现对字典中元素的高效压缩

哈夫曼树和编码在数据压缩和通信领域有着重要的应用。通过根据数据的频率分配不同长度的编码，哈夫曼编码能够在保证数据正确性的前提下，实现高效的压缩和传输。
