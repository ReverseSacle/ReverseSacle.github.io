# Graph

# 概念

  • 有很少条边或弧(e <nlognnlognnlogn)的图称为稀疏图(Parse graph)，反之称为稠密图(Dense graph)

  • 图的边(弧)具有与它相关的数，这种与图的边(弧)相关的数叫做权(Weight)，权可以表示一个顶点到另一个顶点的距离或耗费，这种带权的图通常称为网(Network)

  • 子图(Subgraph)是整体图的一部分

  • 顶点的度(Degree)是和顶点相关联的边的数目

    • 度在有向图中还分入度(InDegree)和出度(Outdegree)，有向图的度为入度和出度之和
      • 入度 → 由其他顶点出发指向当前顶点
      • 出度 → 由当前顶点出发指向其他顶点

  • 一个顶点到另一个顶点的路径(Path)是一个顶点序列，即边的集合

    • 第一个顶点和最后一个顶点相同的路径称为回路或环(Cycle)
    • 序列中顶点不重复出现的路径称为简单路径

  • 在无向图中，如果从一个顶点到另一个顶点有路劲，则称当前顶点和另一个顶点是连通的。如果对于途中任意两个顶点都是连通的，则称当前图为连通图(Connected Graph)

    • 如果是非连通图也有连通分量(Connected Component)，指的是无向图中的极大连通子图

  • 在有向图中，在如果从当前顶点到另一个顶点有路径并且从另一个顶点到当前顶点也有路径，则称当前图为强连通图

    • 有向图中的极大强连通子图称为有向图的强连通分量

  • 一个连通图的生成树是一个极小连通子图，它含有整个图的顶点，但只有足以构成一颗树的n−1n - 1n−1条边

    • 如果在一颗树上添加一条边，必定构成一个环
    • 有n−1n - 1n−1条边的图不一定是生成树

  • 如果一个有向图，恰好有一个顶点的入读为0，其余顶点的入度均为1，则此图是一颗有向树

  • 一个有向图的生成森林由若干棵有向树组成，含有整个图的顶点，但只有足以构成若干棵不相交的有向树的弧

# 十字链表

  • 针对有向图，顶点的结构里有入度和出度指针
  • 指针横纵交叉最终形成一个一个跟十字路口一样，其中有横向的指针也有纵向的指针，所以叫做十字链表

# 邻接多重表

  • 针对无向图，每一条边用一个结点表示，边结点里面有标记信息

# 连通图的遍历

  • 遍历 → 保证每个顶点都被访问到，但其中每一个顶点只能访问一次
  • 为了避免同一顶点被多次访问，在遍历图的过程中，必须记下每一个已访问过的顶点

# 深度优先搜索(Depth First Search)

  • 类似于树的先序遍历
  • 可以从任意顶点出发，都可以把整个图访问完成
