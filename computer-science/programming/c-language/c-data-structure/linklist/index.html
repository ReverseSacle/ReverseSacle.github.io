# 单链表(Singly Linked List)

# 单向链表

单链表是一种基本的线性链式数据结构，由一系列节点组成，每个节点包含数据元素和一个指向下一个节点的引用(指针或链接)。链表的最后一个节点的引用通常为NULL，表示链表的末尾。

基本特性

  • 节点结构 - 每个节点包含两个部分，一个存储数据的字段和一个存储指向下一个节点的指针字段
  • 头指针 - 链表的第一个节点称为头节点，而头节点的指针称为头指针，通过头指针可以访问整个链表
  • 动态大小 - 链表的大小可以动态调整，通过在运行时添加或删除节点来实现
  • 非连续存储 - 节点在内存中不必连续存储，与顺序表不同

# 静态单向链表

由于C语言中有指针操作的概念，其能直接操作计算机内存，但有些编程语言没有指针的概念，倘若想在没有指针操作下创建一个链表，只能通过对链表的概念进行模拟来实现。

静态单向链表是一种链式存储结构，它与动态单向链表的主要区别在于内存分配方式。在静态单向链表中，内存空间是在编译时就分配好的，而在动态单向链表中，内存空间则是在运行时动态分配的。

基本特性

  • 静态分配内存 - 通常通过数组来实现，每个节点使用数组的一个元素，内存空间在编译时就被分配好了
  • 固定大小 - 静态单向链表的大小在编译时就被确定了，不能动态改变
  • 节省指针空间 - 与动态单向链表相比，静态单向链表不需要额外的指针空间来存储下一个节点的地址，因为节点的位置是通过数组下标来确定的

# 单向循环链表

单向循环链表是一种特殊的链表，其最后一个节点的指针域指向头节点，形成一个闭环。与普通单向链表不同的是，在单向循环链表中，任意一个节点都有后继节点，而最后一个节点的后继节点是头节点。

基本特性

  • 闭环结构 - 最后一个节点的指针域指向头节点，形成一个闭环，使得链表首尾相连
  • 无NULL指针 - 节点的指针域不为NULL，而是指向下一个节点或头节点

# 双链表(Doubly Linked List)

# 双向链表

双链表是一种链表数据结构，每个节点除了包含指向下一个节点的指针外，还包含指向前一个节点的指针。这使得在双链表中可以从任意一个方向(前向或后向)遍历节点，相较于单链表，双链表的操作更加灵活。

# 双向循环链表(Doubly Circular Linked List)

双向循环链表是一种链表数据结构，它是双链表和循环链表的结合，每个节点既包含指向下一个节点的指针，也包含指向前一个节点的指针。与双向链表类似，双向循环链表的最后一个节点的后继节点是头节点，形成一个闭环。

# 跳跃表(Skip List)

跳跃表是一种用于维护有序元素集合的数据结构，它通过层级结构实现快速的搜索、插入和删除操作。跳跃表最初由William Pugh在1989年提出，它的设计灵感来自于平衡树，但相对于平衡树，跳跃表的实现更加简单，并且在实际应用中具有较好的性能。

基本的跳跃表结构

  • 节点(Node) - 跳跃表的基本元素，包含一个或多个数据元素和指向下一层节点的指针

  • 层级索引(Level Index) - 每个节点可能包含多个指向下一层节点的指针，这构成了跳跃表的多层结构

跃表的特点

  • 跳跃表通过在原始链表上建立多层索引，每一层都是原链表的一个子集。这些层级的节点称为"跳跃"层，通过这些层级可以更快地定位到目标元素
  • 从底层到顶层，每一层的节点包含的元素数量逐渐减少，层级越高，跳跃的间隔越大。这样的设计使得跳跃表在进行查找时，可以通过不断向右和向下的移动，从而快速定位到目标元素
  • 跳跃表保持一定的平衡性，每个元素都有相同的概率出现在不同的层级上，从而避免了平衡树中需要频繁的平衡操作的复杂性
