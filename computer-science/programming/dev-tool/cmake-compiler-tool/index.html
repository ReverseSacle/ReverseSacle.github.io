# 概述

CMake是一个跨平台的项目构建工具，能解决Makefile对编译平台的依赖性、项目依赖关系容易出错与编写Makefile文件工作量大的问题。

→CMake官方文档←

# 创建

新建一个CMakeLists.txt文件，该文件的名字与大小写不能变。

CMake支持大写、小写、混合大小写的命令。

# 注释

行注释 - # 注释内容

块注释 - #[[ 注释内容 ]]

# 相关命令

编译的时候，需要通过调用本地的cmake工具来完成的，该工具有对应的版本。

cmake_minimum_required() - (可选，不加会⚠)指定使用的cmake的最低版本，对使用的cmake工具版本进行判断，不同版本的cmake工具的命令可能不同

project() - 定义项目工程的信息

  • 名字
  • 工程的版本
  • 工程的描述
  • 工程的web主页地址
  • 工程所支持的语言

add_executable() - 为项目工程生成一个可执行程序

  • 指定可执行程序的名字
  • 所有的源文件名

aux_source_directory() - 查找某个路径下的所有源文件，包括.c/.cpp

  • 第一个参数为要搜索的路径
  • 第二个参数为将要搜索的路径的源文件储存到该变量中

set(变量名 值 [可选参数]) - 定义变量，值类型默认为字符串，需其他类型需额外变更类型

file(GLOB/GLOB_RECURSE 变量名 要搜索的文件路径和文件类型) - 文件的搜索命令

  • GLOB - 将指定目录下搜索到的满足条件的所有文件名生成一个列表，并将其储存到变量中
  • GLOB_RECURSE - 递归搜索指定目录，将搜索到的满足条件的文件名生成一个列表，并将其储存到变量中
  • 变量名
  • 要搜索的文件路径和文件类型

include_directories(path) - 包含头文件，推荐决定路径

set(变量名 ${变量1} ${变量2} ...) - 进行字符串拼接

list(APPEND 储存变量名(list) 需拼接的所有字符串变量) - 进行字符串拼接，APPEND指执行拼接命令，拼接的字符串以;分隔，一个或多个

list(LENGTH 需查询的变量名(list) 储存变量名) - 获取list的长度，以;来定

list(GET 需查询的变量名(list) 索引值 储存变量名) - 获取对应索引的值，索引可为负数

list(FIND 需查询的变量名(list) 需搜索的内容 储存变量名) - 返回索引，未找到返回-1

list(JOIN 需重新拼接的变量名(list) 指定拼接用的连接符 储存变量名) - 将字符串重新用指定的符号进行拼接

list(INSERT 需被插入的变量名(list) 索引 需插入的所有字符串变量) - 在list中指定索引位置插入元素，一个或多个

list(PREPEND 需被插入的变量名(list) 需插入的所有字符串变量) - 在list中索引值为0的位置插入元素，一个或多个

list(POP_BACK 需被操作的变量名(list) 被移除的值的储存变量名) - 将列表最后一个元素移除

list(POP_FRONT 需被操作的变量名(list) 被移除的值的储存变量名) - 将列表第一个元素移除

list(REMOVE_ITEM 需被操作的变量名(list) 值) - 将指定的元素从列表中移除

list(REMOVE_AT 需被操作的变量名(list) 索引值) - 移除指定索引位置的元素

list(REMOVE_DUPLICATES 需被操作的变量名(list)) - 移除列表中的重复元素

list(REVERSE 需被操作的变量名(list)) - 将列表内容翻转

list(SORT 需被操作的变量名(list) 排序规则 大小写规则 升序或降序)

  • 排序规则
    • STRING - 按字母顺序，默认方法
    • FILE_BASEANME - 如果是一系列路径名，会使用basename进行排序
    • NATURAL - 按照自然数顺序
  • 大小写规则
    • SENSITIVE - 按照大小写 敏感 的方式进行排序，默认值
    • INSENSITIVE - 按照大小写 不敏感 的方式进行排序
  • 升序或降序
    • ASCENDING - 按照升序进行排序，默认值
    • DESCENDING - 按照降序排序

-D - 指定宏，设置路径时建议使用决定路径

add_definitions(-D宏名称) - 定义一个宏

# 通过CMake制作库文件

静态库的名字分为三部分：lib, 库名字, .a/.lib，动态库只有后缀名不同。

动态库也称共享库，多个进程可以使用同一份动态库内存中的函数，每个进行使用函数时会映射到内存中各自的虚拟地址空间，mmu会把虚拟地址空间数据映射到物理内存中另外一块地址中，每个进程都对应一块不同的物理内存地址，操作后的数据会被记录到不同的进程的对应物理内存地址中，不会造成数据混乱。

add_library(库名称 STATIC/SHARED 所有源文件) - 制作库命令

  • 库名称 - 指静态库名字三部分的中间的库名字部分
  • STATIC/SHARED- 指定库类型，当前是指静态库(STATIC)/动态库(SHARED)

# 链接库文件

需要库文件与头文件。

link_directories() - 链接动态库或静态库所在路径命令，一个或多个，空格分隔

# 链接静态库

link_libraries(所有静态库名字) - 链接静态库命令，一个或多个，空格分隔，可以全名也可去除后缀，一般用于系统库，找不到自定义的库

target_link_libraries(target 需加载的静态库) - 链接静态库命令

可执行程序没有调用动态库，则动态库不会被加载到内存中。

# 链接动态库

target_link_libraries(target PRIVATE|PUBLIC|INTERFACE 需加载的动态库) - 链接动态库命令，一个或多个，空格分隔

  • 动态库链接具有传递性 - 如果动态库A链接的动态库B，C，动态库D链接了动态库A，此时动态库D相当于也链接了动态库B，C，并可以使用动态库B，C中定义的方法

  • target - 可以是一个链接，一个动态库文件，一个可执行文件

  • PRIVATE|PUBLIC|INTERFACE - 动态库的访问权限，默认PUBLIC，可忽略

    • 由于有传递性，但是由于访问权限的存在，可以对第三方隐藏相应内容
    • 各动态库之间没有依赖关系，不需要设置，直接默认
    • PUBLIC - 在PUBLIC后面的库会被LINK到前面的target中，里面的符号也会被导出并提供给第三方使用。传递性 - 调用函数时会知道调用了哪个库的函数
    • PRIVATE - 在PRIVATE后面的库会被LINK到前面的target中，后边终结掉，第三方获取不到。传递性 - 只能传递一次并且是就近原则
    • INTERFACE - 在INTERFACE后面的库不会被LINK到前面的target中，只会导出符号。传递性 - 只能知道有一个函数但是不知道是属于哪个库的

  • 需加载的动态库

# 打印日志

message([] 消息内容) - 显示一条消息

  • []为空 - 重要消息
  • []为STATUS - 非常重要的消息
  • []为WARNING - CMake警告，会继续执行
  • []为AUTHOR_WARNING - CMake警告(dev)，会继续执行
  • []为SEND_ERROR - CMake错误，继续执行，但是会跳过生成的步骤
  • []为FATAL_ERROR - CMake错误，终止所有处理过程

CMake的命令行工具会在stdout上显示STATUS消息，在stderr上显示其他所有消息。CMake的GUI会在它的log区域显示所有消息。

CMake的警告和错误消息的文本显示使用的是一种简单的标记语言，文本没有缩进，超过长度的行会回卷，段落之间以新行做为分隔符。

# CMake嵌套

每个项目都可能有各自的CMakeLists.txt文件，嵌套的CMake是一个树形结构。

  • 根节点CMakeLists.txt文件 - 变量全局有效
  • 父节点CMakeLists.txt文件 - 变量可以在子节点中有效
  • 子节点CMakeLists.txt文件 - 变量只能在当前节点有效

add_subdirectory(源所在目录) - 添加子目录
