# 概叙

在C/C++程序中，经常需要调用其他的程序来预先完成某项任务，例如其他的C/C++程序，操作系统命令或Shell脚本，C/C++提供了exec函数族和system函数来实现这个功能。

# exec函数族

exec函数族提供了一个在进程中启动另一个程序执行的方法。它可以根据指定的文件名或目录名找到可执行文件，并用它来取代原调用进程的数据段，代码段和堆栈段，在执行完后，原调用进程的内容除了进程号外，其他全部被新的进程替换了。这里的可执行文件即可以是二进制文件，也可以是Linux下任何可执行的脚本文件。

  • exec函数族的声明

  • 参数解析

    • path - 要执行的程序路径，可以是绝对路径和相对路径。在execv,execl和execle这三个函数中，使用带路径名的文件名作为参数
    • file - 要执行的程序名称。如果该参数中包含/字符，则视为路径名直接执行；否则视为单独的文件名，系统将根据PATH环境变量指定的路径顺序搜索指定的文件
    • arg - 程序的第0个参数，即程序名自身。相当于arg[0]
    • argv - 命令行参数的数组
    • envp - 带有该参数的exec函数可以在调用时指定一个环境变量数组。其他不带该参数的exec函数则使用调用者进程的环境变量
    • ... - 命令行参数列表。调用相应程序时有多少命令行参数就需要有多少个输入参数项
    • 返回值 - 执行成功函数不返回，执行失败则直接返回-1，失败原因存储在errno中

  • 在使用此类函数时，在所有命令行参数的最后应该增加一个空的参数项(NULL)，表明命令行参数结束

  • 在主程序中成功调用execl后，被调用的程序将取代调用者程序，即execl函数之后的代码都不会被执行

  • 实际开发中，最常用的是execl函数，其他极少使用

# system函数

  • 提供了另一种简单的执行程序的方法，把需要执行的命令用一个参数传给system函数
  • 函数声明 - int system(const char* string)
  • system会调用fork函数产生子进程，由子进程来调用/bin/sh -c string来执行参数string字符所描述的命令，此命令执行完后随即返回原调用的进程，在调用system期间SIGCHLD信号会被暂时搁置，SIGINT和SIGQUIT信号则被忽略
    • 如果fork执行失败，返回-1，并出现错误
    • 如果execl失败，表示不能执行shell，返回值相当于shell执行了exit(127)
    • 如果执行成功，则返回子shell的终止状态
    • 如果system在调用/bin/sh时失败则返回127，其他失败原因返回-1
    • 若参数string为空指针(NULL)，仅当命令处理程序可用时，返回非零值
    • 如果system调用成功则最后会返回shell命令后的返回值，但是此返回值也由可能为system调用/bin/sh失败所返回的127，因此最好能再检查errno来确认执行成功
