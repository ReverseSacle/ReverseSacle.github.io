# 安装

yum install gdb -y

# 基本命令

  • gdb 可执行程序名 - 进入gdb调试模式

  • 进入gdb模式后的命令

    • start - 使程序开始运行并开始调试程序

    • b 断点位置 - (break)如b 10，将断点位置设置在代码第10行，可设置多个断点

    • r - (run)开始运行程序，遇到断点会停下，没有断点就一直运行到结束

    • n - (next)执行当前代码行的语句(类似visual studio的F10)

    • s - (step)执行当前代码行的语句，是函数则进入到函数内部，库函数无法进入(类似visual studio的F11)

    • c - (continue)继续运行程序，直到遇到下一个断点

    • p 变量名 - (print)打印变量的值

    • set var 变量名=值 - 设置程序已声明变量的值

    • set 参数值1 参数值2... - 设置主程序函数参数

    • list - 打印源代码，回车下一步

    • layout src - 原代码可视化

    • layout asm - 原代码汇编形式的可视化

    • q - (quit) 退出gdb模式

# 调试core文件

使用-g指令后，当程序运行出错后，操作系统会生成一个core文件，该文件包含程序在内存里的执行到出错时的执行信息。

系统默认core文件太大时，不会生成该文件。

命令

  • ulimit -a - 查看系统参数

  • ulimit -c unlimited - 将core文件的允许大小提高至无限制

  • gdb 可执行程序名 core文件名 - 进入gdb模式

    • bt - 查看函数的调用栈

# 调试正在运行的程序

第一个客户端窗口运行程序，在第二个客户端窗口调试，第一和第二各客户端连接同一个地址。

命令

  • ps -ef | greap 第一个窗口运行中的程序名 - 查看当前运行程序的进程
  • gdb 第一个窗口运行中的程序名 -p 进程编号 - 进入gdb调试，程序被暂停执行，退出后会继续执行

# 调试多进程的服务程序

gdb模式下默认调试父进程，子进程保持运行。

命令

  • ps aux | grep 可执行程序名 - 查看当前运行程序的进程

  • gdb 可执行程序名 - 进入gdb调试模式

    • set follow-fork-mode parent - 声明调试父进程
    • set follow-fork-mode child - 声明调试子进程
    • set detach-on-fork on或者off - on表示缺省，默认调试当前进程的时候，其他进程继续运行；off模式下，调试当前进程的时候，其他进程被gdb挂起，子进程不会执行。
    • info inferiors - 查看可调试的全部进程
    • inferior 进程编号 - 切换当前调试进程

# 调试多线程的服务程序

命令

  • ps -aL | grep 可执行程序名 - 查看当前运行程序的轻量级进程(线程)
  • pstree -p 线程编号 - 查看主线程与子线程的关系
  • gdb 可执行程序名 - 进入gdb调试模式
    • info threads - 查看当前的所有线程(*表示当前线程)
    • thread 线程Id - 切换调试线程
    • set scheduler-locking on - 声明只运行当前线程，其他线程挂起
    • set scheduler-locking off - 取消set scheduler-locking on的声明
    • thread apply 线程Id gdb命令 - 指定某个线程执行某个gdb命令
    • thread apply all gdb命令 - 使全部线程执行某个gdb命令
