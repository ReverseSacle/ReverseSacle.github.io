# 概述

C/C++一般可用GDB调试，或者把程序运行的情况，数据或异常等信息输出到屏幕上。

对于后台的服务程序，没有操作界面，程序运行的状态，数据处理的日志，程序的异常等必须记录在日志文件中，运维人员根据日志文件的内容，查看程序运行和数据处理的情况。

# 多进程与多线程服务程序的日志

在多进程和多线程服务程序中，使用同一日志文件有可能会产生竞争。

多进程程序的日志文件不能切换。

如果使用同一日志文件，用锁太麻烦，还会影响程序的性能，一般分开写日志。

# 第三方日志库

spdlog官方文档

spdlog是一个快速、轻量级的C++日志库，具有异步日志记录、多线程支持和灵活的日志格式化选项。

支持多种后端，包括控制台、文件和系统日志等；易于集成，具有高性能。

Pantheios官方地址

Pantheios是一个C++的跨平台日志库，旨在提供高性能、低延迟的日志记录功能。它的设计理念包括极低的运行时开销、灵活的日志格式化选项以及对多种后端目标的支持。
