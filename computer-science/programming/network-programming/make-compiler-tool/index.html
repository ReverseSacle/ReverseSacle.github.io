# 叙述

make是一个程序，用来管理多模块程序的编译的链接，直接生成可执行文件。

make程序需要一个编译规则说明文件，称为makefile。

  • makefile文件中描述了整个软件工程的编译规则和各个文件之间的依赖关系
  • makefile像shell脚本，可执行操作系统命令，能实现自动化编译

make是一个解释makefile中指令的命令工具，使用make可以使重新编译次数达最小化。

安装命令 - yum install make

# 规则详解

# 基本规则

  • all标签 - 表示有多少个目标需要编译，并指明了makefile中哪些标签名称下的命令会在执行make命令时自动执行。\表示换行

  • xxx1标签是所需编译目标的其中一个，此处指定了xxx1。表示编译xxx1需要依赖xxx1.c文件，并执行编译命令。其中，shell命令前面需要按下Tab键，不能是空格

  • 同上，可以执行多条命令

  • 同上，表示编译xxx2需要编译xxx2.c，_pb.h，_pb.c三个文件，并执行编译命令

  • 当代码文件内容没有改变，需编译的文件数量没有变动时，使用make时不会重新编译

  • makefile中可能会有clean标签，它没有任何依赖。由于clean标签不在all标签中，因此在执行make命令时不会自动执行，使用时需手动输入命令make clean。此外，通过make 标签名命令还能执行指定标签的命令

  • 在标签下面除了可以是编译命名外，还可以是操作系统的命令

# 使用变量

  • makefile中变量是一个名字，它的值就是一个文本字符串
  • 在makefile中，依赖，命令或任何引用变量的地方，都会被变量代替。有点像C语言的宏
  • 使用变量的方法 - $(变量名)
