# FiddlerEverywhere

→Fiddler下载地址←

# 设置

开启HTTPS抓包：界面 → 右上角齿轮Settings → HTTPS → 点击Trust Fiddler CA → 接受 → 勾选Capture HTTPS traffic与Ignore server certificate errors (unsafe) → 点击右下角Save

开启远程连接：界面 → 右上角齿轮Settings → Connections → 勾选Allow remote devices to connect → 点击右下角Save

# 连接

电脑与手机都连接同一个WIFI。

界面 → Remote Devices → Android → Next → 右边选择你电脑当前所使用的网卡 → 用手机浏览器扫描QR码 → 手机中点击下载 →

分支一：下载后打开点安装证书手机设置中搜索凭据 →

分支二：找到信任的凭据(CA证书) → 选择安装凭据 → CA证书 → 选择刚刚下载的证书 →

Next → Next → 手机点击已连接的WIFI → 打开已连接的WIFI的设置 → 点击代理选择手动 → 输入Fiddler中Enter the following information中的内容,Server指主机名,Port指端口 → Next → 用手机浏览器扫描QR码 → 如果出现有安全风险就选择接受风险 → Finish

# 抓包

Fiddler中点击左边的眼睛，打开手机APP，配合Fiddler中右边的垃圾桶形状按钮，再进行相应操作，之后就是逆向了。

有些app有加密(例如某自聘)，加密压缩方式不知道是什么，返回内容都是�，即无法解码。而有些app没有加密(例如某聘)，返回的数据在Fiddler中直接被解码了。

# 电脑端

Help → SSL Proxying → Install Charles Root Certificate → 安装证书 → 选当前用户并点下一步 → 将所有的证书都放入下列储存 → 选受信任的根证书颁发机构并确定 → 下一步 → 完成 → 跳出提示点 是

# 手机端

确保手机端与电脑端处于同一WIFI网络中。

Help → SSL Proxying → Install Charles Root Certificate On a Moblie Devices or Remote Browser→ 跳出弹窗 → 弹窗中HTTP proxy on后面的地址为当前主机地址:端口号

手机端打开WIFI设置 → 打开已连接的WIFI的设置 → 找到代理 → 点击选手动 → 将上面的当前主机地址填入主机名一栏中，端口号填入端口一栏中 → 确认 → 返回Charles会出现弹窗 → 选Allow

打开手机浏览器输入chls.pro/ssl → 点继续前往 → 点下载 →

分支一：下载后打开点安装证书手机设置中搜索凭据 →

分支二：找到信任的凭据(CA证书) → 选择安装凭据 → CA证书 → 选择刚刚下载的证书

# 相关配置

Proxy → 点击Windows Proxy

Proxy → Proxy Settings → Proxies → 勾选Enable transparent HTTP proxying

解决抓取内容乱码：Proxy → SSL Proxying Settings → SSLProxying → Include中点击Add → Host填*，Port填443 → 点OK后重启Charles

# 抓包

由于数据中包含电脑端，需用左下角的Filter填入相应关键字来筛选对应的内容
