# 概述

JSON(JavaScript Object Notation - JavaScript对象标记法)，是一种轻量级，基于文本(可用记事本打开)，可读的(人可以阅读)格式，其语法规则是参考JavaScript对象的，其语法与JavaScript定义对象的语法几乎完全相同，并且JSON格式并不是只能用于JavaScript。

JSON格式，对于人和机器都是十分方便阅读的，相比另一种常见的数据交换格式XML，文件更小。

JSON格式的创始人声称此格式永不升级，这就表明该格式具有长时间的稳定性。

# JSON的语法规则

名称(name)/值(value)对之间用冒号:隔开。

  • 名称(name)是一个用双引号括起来的字符串，不能用单引号，也不能没有引号
  • 值(value)的类型只有六种，与JavaScript不同的是，不能有undefined，函数等，只能使用下面六种类型
    • 字符串(string) - 只能用英文双引号括起来，不能用单引号，也不能没有引号。能够使用转移字符，能够输入任意的Unicode格式的字符，除了双引号"
    • 数值(number) - 包含了基本数值类型，除了十六进制和八进制
    • 对象(object)
    • 数组(array)
    • 布尔类型(true/false)
    • null

对象(Object)用大括号{}表示，大括号里是一系列的名称/值对。

数组(Array)用方括号[]表示，方括号内可以有一系列不同的值(例如，基本数据类型，数组，对象等)。

并列的数据之间用英文的逗号,分隔，最后一个末尾不用加逗号。

JSON与XML的区别？

JSON相较于XML的优势

  • 没有结束标签，长度更短，读写更快
  • 能够直接被JavaScript解释器解析
  • 可以适用数组

上例描述了在相同的信息，XML的信息量比JSON大得多，尤其是当有数组出现时，这更能凸显JSON格式的优势。

# JSON的解析和生成

将符合JSON语法规则的字符串转换成对象的这个过程，称为解析。JavaScript中主要有三种解析方法

  • eval()函数 - 参数是一个字符串，作用是直接执行其中的JavaScript代码。能够用于解析JSON字符串，但很少使用，只有当浏览器版本很低时才使用。其也是一个相对危险的函数，因为字符串中可能含有未知因素

  • JSON.parse()方法 - 推荐方法，如果输入了不规范的字符串时会报错，有两个参数，第一个参数是要解析的字符串；第二个参数是函数，该函数有两个参数，名称(name)与值(value)，当传入一个JSON字符时，JSON的每一组名称/值对都会调用此函数，函数的返回值会赋值给当前的名称(name)

  • 第三方库(例如JQuery)

将一个JavaScript对象转化为JSON字符串的这个过程，称为序列化。

主要用JSON.stringify(value,replacer,space)方法生成JSON，该函数有三个参数。

  • value - 必填参数，被转换的JavaScript值，一般是对象或数组

    • 对象里面不符合规则的内容会自动被剔除
    • 数组里面不符合规则的值会变为null

  • replacer - 可选参数，有两种选择

    • 函数 - 每一组名称/值对都会调用此函数，该函数返回一个值，该值作为名称的值变换到结果字符串中，如果返回undefined，则该成员被忽略

    • 数组 - 只有数组中存在名称才能被转换，并且转换后顺序与数组中的值保持一致

  • space- 可选参数，为了排版，方便阅读而存在，可以在JSON字符串中添加空白或制表符等

# Ajax与JSON

JSON文件被放置在服务器端，客户端请求该文件用得最多的是Ajax，能够实现异步请求。

Ajax(Asynchronous JavaScript and XML)，即异步的JavaScript和XML，一般写作Ajax。

Ajax能够与服务器交换少量数据，从而异步地更新部分文件。

异步，指的是当Ajax执行交换数据的操作时，其他的操作仍然可以执行。一个常见的应用是，打开百度或谷歌首先，输入文字在搜索栏，搜索栏下方会显示出几个建议的搜索词。

# GeoJSON与TopoJSON

GeoJSON与TopoJSON不是新的格式，而是符合JSON语法规则的两种数据格式，用于表示地理信息。

# GeoJSON

GeoJSON是用于描述地理空间信息的数据格式，专门用于表示地理信息，其最外层是一个单独的对象，这个对象可用于表示

  • 几何体(Geometry)
  • 特征(Feature)
  • 特征集合(FeatureCollection)

最外层的GeoJSON里可能包含很多个子对象，每一个GeoJSON对象都有一个type属性，用于表示对象的类型，type值必须是下面其中一个

  • Point - 点
  • MultiPoint - 多点
  • LineString - 线
  • MultiLineString - 多线
  • Polygon - 面
  • MultiPolygon - 多面
  • GeometryCollection - 几何体集合
  • Feature - 特征
  • FeatureCollection - 特征集合

每一个对象都有一个coordinates成员变量。如果type值为Point, MultiPoint, LineString, MultiLineString, Polygon, MultiPolygon其中之一，则该对象必须有coordinates成员变量。

如果type值为几何体集合(FeatureCollection)，那么该对象必须有变量geometries，其值是一个数组，数组的每一项都是一个GeoJSON的几何对象，例如

如果type值为特征(Feature)，那么此特征对象必须包含有变量geometry，表示几何体，geometry的值必须是几何体对象。此特征对象还包含有一个properties，表示特性，properties的值可以是任意JSON对象或null，例如

如果type值为特征集合(FeatureCollection)，则该对象必须有一个名称为features的成员。features的值是一个数组，数组的每一项都是一个特征对象。

# TopoJSON

TopoJSON是GeoJSON按拓扑学编码后的扩展形式，是由D3的作者Mike Bostock制定的。相比GeoJSON直接使用Polygon, Point之类的几何体来表示图形的方法，TopoJSON中的每一个几何体都是通过将共享边(称为arcs)整合后组成的。

TopoJSON消除了冗余，文件大小缩小了80%，因为，边界线只记录一次(例如广西与广东的交界线只记录一次)，地理坐标使用整数，不使用浮点数。
