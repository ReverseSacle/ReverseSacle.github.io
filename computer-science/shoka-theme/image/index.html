Vercel反代Jsdelivr加速链接

须知前提

  • 由于国内政策而使Jsdelivr的服务政策发生变动，国内ip访问加速链接时，所有图片、视频等画面输出类文件(.ico文件除外)的加速地址都不会被cdn加速，而是指向默认的Github源
  • 对于Vercel官方免费提供的资源(每月免费100G)，其存在一定的限制，具体情况可至官方查阅

准备条件

  • 一个域名(可选但推荐)
  • Linux系统(虚拟机)/bash终端/Windows系统等
  • 梯子(依情况而定，可能用不到)
  • Vercel账号

正式开始

  • 以Windows10系统为例

  • (默认已安装npm)先确保已安装了Node.js，之后，在windows桌面上新建一个文件夹，在文件夹中的左上的路径栏中输入cmd并回车，待出现命令符窗口后，输入命令npm install -g vercel --registry https://registry.npmmirror.com(有点慢，要是特别慢就换梯子)

  • 之后，输入vercel login，这里选择Continue with GitHub(选择的位置以等待输入光标为准，Enter键确认，具体选择依自身情况而定)，之后会跳出登入页面

  • 返回到刚刚创建的文件夹，新建一个名为cdn.jsdelivr.net的文件夹(名字自定义)，进入该文件夹，新建一个cdn.jsdelivr.net.json文件(名字自定义，该文件的后缀是.json)，用文本编辑器(我用的是Notepad++)打开cdn.jsdelivr.net.json文件并输入下面的反代代码

  • 最后在cdn.jsdelivr.net文件夹的路径栏中输入cmd并回车，终端输入部署命令vercel -A cdn.jsdelivr.net.json --prod

  • 注意what's your project's name?这个提示，此处的提示是要求你输入部署项目的名称(按喜好来定就行了，另外这里不支持特殊字符，-号除外)，其他的选项可依照下方图片提示来选择即可

  • 最后，进入Vercel官网，登入后可以看到刚刚创建的项目，点击项目，进入后记录下DOMAINS处的链接地址(其中一个即可)

  • (可选)由于Vercel有时也在墙内，可能需要绑定子域名才可顺畅使用。在Vercel官网，点击刚刚创建的项目→右上角的Domains→Edit→Domain处改为自己的子域名→点击Save，之后再在域名购入商处添加值为(cname.vercel-dns.com或cname-china.vercel-dns.com)的CNAME类型记录

关于PicGo

info
PicGo是目前主流图床的集合API，支持SM.MS，腾讯云COS，Github，七牛，Imgur，阿里云OSS，又拍云等图床，另外PicGo还可自行安装其他图床插件。注意，上传区中的URL链接格式不可使用
将图片托入即可上传，也可批量托入，可上传各种图片格式，gif格式也可以；上传后在相册中查看。对于上传速度慢或无法上传的问题，可开启科学上网工具(梯子)

 

(可选一)Github图床 

  • (评价)稳定性一般，但简易便捷，墙外速度一流，墙内无速度可言，零成本

  • 先在Github中新建一个公开的仓库

  • 之后点击Github头像→Settings→侧栏Developer settings→Tokens(classic)→Generate new token(classic)

  • token例子，只选repo即可；其中Expiration为凭证到期时长，当过期时可在此凭证中点击Regenerate token重置时长。之后可以在此页面看到新添加的token以及一串编码，此编码就是token，此外这个token编码只能看到这一次，记得保存

  • 弄好上述步骤后，安装PicGo，-x64.exe为64位系统，-ia32.exe为英特尔32位系统。安装后打开，不会弹出主界面，要在系统托盘处双击打开；(可选)在PicGo中点击PicGo设置将时间戳重命名打开(上传的图片会以时间戳的形式重命名)，其他设置可自行按需配置

  • 之后点击图床设置找到GitHub图床(不支持相册与Github仓库的同步删除)

  • (可选)推荐使用github-plus插件代替初始的Github图床插件，配置完全一致，但此插件支持同步PicGo内相册处的删除功能(例如，PicGo相册内删除某一张图片，对应Github仓库内也会删除对应的图片(可在PicGo的插件设置处搜索安装)

  • 图床配置例子

    info
    设定仓库名为Github用户名/仓库名；设定自定义域名推荐使用Jsdelivr的CDN加速链接 (https://cdn.jsdelivr.net/gh/你的Github用户名/Github仓库名@main)
    指定存储路径指你新建仓库里的文件夹名，没有会自动创建一个新的指定文件夹并在里面存放图片

 

(可选)反代Jsdelivr图片链接

  • 由于Jsdelivr不再提供大陆地区的所有图片和视频等文件类型的加速，图床外链的CDN加速效果已失效，当使用图床外链时会被强制指向Github源。但是，倘若你拥有一个域名，可用下面介绍的方法解决

  • 请先阅览上一个标签块Jsdelivr引起的国内访问图片缺失问题(墙内)，阅览后将设定自定义域名中的cdn.jsdelivr.net改为你的反代链接(DOMAINS链接地址)即可

前往腾讯云COS 对象存储官网地址

阿里云OSS 对象存储

前往阿里云OSS 对象存储官网地址

  • 阿里云OSS图床同理，此处不作介绍，推荐先看后面的注意项及操作提示

  • (评价)需一定成本，墙内外速度偏向一流，操作较为繁琐，但稳定性高，安全性强

  • (注意)由于自建图床存在成本和安全性的问题，在创建图床之前应先计算下成本以及了解其安全设置的繁杂情况

  • →COS计算费用地址

  • 还有一种预付费计费模式，就是先买流量包和存储空间包，对象存储耗费的在购买的包内扣除，同手机电话卡一样的模式

正式操作

  • 先注册腾讯云，还需完成实名验证，之后再开通对象存储。点击对象存储控制台左侧的存储桶列表→创建存储桶

  • 所属地域选择靠近你博客服务器地址的地域；名称推荐填字母(填中文解析链接时常比较久)，记得避免后面一串数字泄露造成隐私泄漏的问题；(可选)高级可选配置中只将服务端加密打开即可，其他打开需额外费用

  • 之后，在存储桶名称→点击相应储存桶名称→安全管理→防盗链设置→白名单→空referer处先选择允许，Referer处填你的域名地址，例如*.example.com(有个人域名的可再填入域名的ip地址)

PicGo相关配置

  • 为了防止隐私泄露，需要一个子账号，该子账号只拥有对COS的读写、请求访问等功能即可。点击右上角头像图标→账号信息→页底的快速创建子用户

  • 创建完成后，点击右上角头像图标→访问管理→左边的用户列表→刚创建的子用户→点击操作处的授权，搜索栏搜索并勾选QcloudCOSDataFullControl权限和QcloudCOSFullAccess权限。授予权限完成后，点击子用户的用户名称，进入用户详情页面后点击API密钥中的新建密钥

  • 再在PicGo的图床设置中选择腾讯云COS，将SecretId和SecretKey填入，之后返回腾讯云页面，点击左上角云产品→选择对象存储→左边的存储桶列表。返回PicGo，APPID为存储桶名称-后面一串数字，存储空间名为存储桶名称，存储区域可在存储桶列表页面的所在区域处查看(只填入括号内的英文即可)

  • 最后就是图片的上传，处理完后在腾讯云COS存储桶→安全管理→防盗链→将空referer设为拒绝

 

注意项及操作提示

warning
1.先吐槽一下，不管是注册腾讯云还是阿里云都会有客服电话，挺烦的，直接挂断拉黑了，倘若硬是要接的话，就说不需要该电话服务。另外，倘若不想实名注册的，可用国际版的腾讯云或阿里云，但需要港/澳/台或国外的手机号。
2.推荐先参加腾讯云的答题(免费领3个月的免费额度)，3个月后再将图床转移到阿里云OSS(新用户有1元3个月的免费额度，但仅限国内地区的bucket)。图床规模很大的用户不推荐，毕竟迁移很费时间，倘若只是将腾讯云COS/阿里云OSS当作图床，那这两个的费用差距不大。另外再吐槽下，两个对象储存都实际体验了三个月，发现腾讯云COS链接产生的请求数会明显多于阿里云OSS链接产生的，即使腾讯云COS有免费资源包，但每个月仍有一元以内的消费，而阿里云OSS没有使用免费的资源包，但一个月内的使用都未产生耗费。
3.通过PicGo上传或自己手动上传都会产生对存储桶的请求，此外，任何在存储桶上的操作(比如，存储桶的访问，图片等文件的添加/删除/配置更改等)都会产生请求，这些请求也记入到本月总请求数中。PicGo内的删除操作不会与腾讯云COS同步，这意味着图片的删除需手动进行。当自己网络ip不在存储桶所在区域范围内时，图片的上传/删除才会产生外网下行流量。开启CDN加速后会产生额外流量，这些流量不记入到外网下行流量，但额外参与计费。全球加速费用高，一般情况不推荐开启，只有使用给定的加速链接才会产生耗费，倘若只是用于特定的图片，还是比较推荐。
4.当PicGo指定存储路径不存在时会自动创建，可通过修改图床设置中的存储路径来将图片存放在不同文件夹中；当使用PicGo进行图片操作时记得将腾讯云COS存储桶→安全管理→防盗链→空referer设为允许，方便查看。
5.(此处不包括PicGo的操作)每当有腾讯云COS内部操作需要短暂需要密钥的时间来操作时，推荐先创建子用户再授予相应权限后再进行相关操作，最后再删除子用户，可以有效避免隐私泄露；当不进行图片操作时，记得将腾讯云COS存储桶→安全管理→防盗链→空referer设为拒绝，否则有可能被其他人用脚本盗取图片链接而产生额外请求量和外网下行流量。(注意)腾讯云COS中不论是主站号还是子账号，它们的密钥经过一段时间后会被刷新，当你发现PicGo上传图片时卡住很久，这就表明需要更换密钥了。
6.当利用hexo s命令来本地预览时，可将腾讯云COS存储桶→安全管理→防盗链→空referer设为允许即可；阿里云OSS的则需在数据安全→防盗链→Referer处添加http://localhost:4000/。
关于图片大小及其加载速度问题

图片压缩

Squoosh官网

前往Squoosh官网地址

  • 假如你原来的图片是超高清图片，当打开你的主页时，顶部的图片播放以及文章封面的加载会因网速限制而不时加载失败或等待时间过长。因此最好将你的图片进行压缩，压缩后视觉观感质量并不会下降多少。单张图片压缩推荐谷歌出品的Squoosh；此外，批量压缩和Gif压缩就没有可推荐的了，目前对于批量压缩和Gif压缩的问题，不同的产品都有各自的优缺点和不适感，若想使用可自行搜索

  • (注意)压缩格式中目前还不太推荐AVIF，即便是最好之一的图片格式，但还存在浏览器的适配问题

  • 此外，倘若你是用CDN加速链接或自费图床来作图床链接的用户，更应该压缩你的图片，通过压缩图片能在达到损耗少量图片质量的前提下大大降低图片的存储大小，可以有效提高图片加载速度和节省图床使用流量。

  • 这里推荐压缩格式为Webp，将图片分辨率定格在2k或2k以下 (Width <= 2560|Height <= 1440，实际还得看自身要求)，压缩后图片大小一般都不会不超过500kb

  • 至于原格式图片要怎样，具体情况依自身而定
